You are building a Neo4j knowledge graph from a structured dataset export.

Your only job in this step is to decide which record types become Neo4j node labels.

EXISTING SCHEMA (reuse these canonical names where applicable):
{shared_context_yaml}

DATASET STRUCTURE SUMMARY:
{structure_summary}

SAMPLE RECORDS:
{sample_records_json}

TASK:
Return ONLY a single JSON object — no explanation, no markdown, no extra text.

The object must have these keys:

- "node_types": for each distinct typeName value in the data, decide:
  1. What PascalCase Neo4j label should it map to?
  2. What field uniquely identifies each record? (almost always "uniqueId")

- "id_field": the field that uniquely identifies each record (e.g. "uniqueId", "id", "uuid")

- "type_field": the field that names the record type (e.g. "typeName", "type", "kind"). null if all records are the same type.

- "hierarchy_config": if the data has a path/breadcrumb field encoding a folder hierarchy, describe it here. Set to null otherwise.

- "nested_collections": array of array-of-objects fields that should become child nodes. Set to [] if none.

Example response:

{{
  "node_types": [
    {{"name": "TestCase", "maps_to": "TestCase", "identity_key": "uniqueId"}},
    {{"name": "XModule", "maps_to": "XModule", "identity_key": "uniqueId"}}
  ],
  "id_field": "uniqueId",
  "type_field": "typeName",
  "hierarchy_config": {{
    "field": "nodePath",
    "separator": "/",
    "phantom_label": "Folder",
    "edge_type": "CONTAINS"
  }},
  "nested_collections": [
    {{
      "field": "details.moduleAttributes",
      "child_label": "ModuleAttribute",
      "edge_type": "HAS_ATTRIBUTE",
      "id_field": "uniqueId"
    }}
  ]
}}

Rules:
- Every typeName in the data must appear in node_types
- Use existing canonical labels from EXISTING SCHEMA where the type is the same concept
- PascalCase labels only (e.g. TestCase, XModule, Folder)
- Do not invent types that are not in the data
- Set hierarchy_config to null if the data has no path/breadcrumb field
- Set nested_collections to [] if no array-of-objects fields should become child nodes
- Do not invent fields that are not present in the sample data
